# Migration Note

기존 `QuantScreen` 원본 레포는 Next.js 프론트와 FastAPI 백엔드가 루트에 혼재된 로컬 개발용 구조였습니다.

현재 대표 레포 구조는 다음을 목표로 재정리했습니다.

- 평가자가 하나의 링크에서 전체 프로젝트를 이해할 수 있게 함
- 기존 정상 배포 환경은 유지해 제출 직전 리스크를 줄임
- 최신 프론트/백엔드 코드를 `frontend/`, `backend/`에 명확히 분리
- 배포 트러블슈팅과 API 계약을 문서화

이 구조는 완전한 운영 전환 이전의 안전한 하이브리드 모노레포 전략입니다.
